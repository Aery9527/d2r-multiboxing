# Plan: Multi-Language (i18n) Support

## 背景與目標

目前全部 ~200 個玩家可見字串均為繁體中文硬編碼。
本計畫為工具加上多語系支援，讓玩家在首次啟動或主選單內自由切換語系。

**初期支援語系**：繁體中文 (zh-TW)、English (en)  
**架構方式**：type-safe nested struct catalog（非 string-key map）

---

## 確認需求

| 項目 | 決策 |
|------|------|
| 初期語系 | zh-TW + en（可擴充） |
| 首次啟動 | 偵測 Windows LCID → 自動預設 → 玩家確認 |
| 切換語言 | 立即生效（reload package-level `lang` var，重繪現有畫面） |
| Catalog 結構 | 巢狀結構體欄位（type-safe, IDE autocomplete） |
| 設定持久化 | `config.json` 新增 `language` 欄位 |

## 架構設計

### 新套件：`internal/common/locale/`

```
internal/common/locale/
  catalog.go      — Locale 型別、所有 *Catalog 巢狀 struct 定義、Get/ParseLocale/KnownLocales
  zh_tw.go        — 繁體中文值
  en.go           — English 值
  detect.go       — DetectSystemLocale() via Windows LCID
  locale_test.go  — 完整性測試 + 行為測試
```

#### Catalog 頂層結構

```go
type Locale string

const (
    LocaleZhTW Locale = "zh-TW"
    LocaleEn   Locale = "en"
)

type Catalog struct {
    Locale   Locale
    Common   CommonCatalog
    Startup  StartupCatalog
    MainMenu MainMenuCatalog
    Launch   LaunchCatalog
    Delay    DelayCatalog
    Switcher SwitcherCatalog
    Flags    FlagsCatalog
    D2RPath  D2RPathCatalog
    Accounts AccountsCatalog
    Language LanguageCatalog
}
```

公開 API：
- `Get(l Locale) *Catalog` — 取得指定語系 catalog（未知語系 fallback zh-TW）
- `ParseLocale(s string) (Locale, bool)` — 驗證 / 正規化語系字串
- `KnownLocales() []Locale` — 列出所有已知語系
- `DetectSystemLocale() Locale` — Windows LCID 偵測，回傳最接近語系

### Config 修改

新增欄位：
```go
// Language is the display language; empty means not yet chosen (first-run).
Language string `json:"language,omitempty"`
```

- `DefaultConfig()` 不預填 Language（空字串 = 首次執行）
- `Load()` 不對 Language 做驗證（empty 合法，由 CLI 層處理）
- 向後相容：舊版沒有 language 欄位的 config 讀取後 Language = ""，觸發首次選擇流程

### CLI 層異動

**新增 `cmd/d2r-hyper-launcher/cli_language.go`**
- `pickInitialLanguage(cfg *config.Config)` — 首次選語言流程
- `setupLanguage(cfg *config.Config)` — 從主選單重新設定

**`main.go`**
- 新增 `var lang *locale.Catalog`（package-level）
- `config.Load()` 後：若 `cfg.Language == ""` → 呼叫 `pickInitialLanguage(cfg)`
- 載入 catalog：`lang = locale.Get(locale.Locale(cfg.Language))`
- `setupLanguage` 儲存後直接 `lang = locale.Get(...)` 立即生效

**主選單新增 `[l]` 選項**（`menu.go` + `main.go` switch）

**所有 `cli_*.go`、`menu.go`、`feedback.go`**  
硬編碼中文字串改為 `lang.Section.Field`，format string 保持 printf 風格：
```go
// 舊
ui.infof("正在啟動 %s (%s)...", acc.DisplayName, region.Name)
// 新
ui.infof(lang.Launch.Starting, acc.DisplayName, region.Name)
```

### 已知限制

`internal/` domain 層的 error string（例如 `config.ParseLaunchDelayRange` 驗證訊息）
本期**不翻譯**。這些 error 值以中文生成，透過 `ui.errorf()` 顯示時仍為中文。
完整 domain 錯誤在地化需要改用 error code 架構，規劃為後續工作。
