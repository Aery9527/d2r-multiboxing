# v1.1.0

## 摘要
- `d2r-hyper-launcher` 這次發行聚焦在多開啟動流程的可讀性、帳號 flag 管理，以及玩家在 CLI 內實際會看到的提示品質。
- 這個版本延續 `v1.0.0` 的正式發行基礎，補齊多帳號啟動、切窗設定、首跑引導與 release 顯示等多個日常使用細節。
- 同步更新 release metadata 與文件入口，讓玩家可以直接在啟動畫面與 README 確認目前執行的是哪個版本、何時 build。

## 重點
- 新增更容易掃描的 per-account `LaunchFlags` 設定流程，並保留目前實際暴露給玩家的兩種旗標選項。
- 單帳號與批次啟動現在會更清楚顯示哪些帳號即將被啟動、哪些帳號其實已經在執行中，以及何時不需要再繼續啟動。
- 啟動畫面現在會直接顯示 release 版本與時間戳，並在首跑需要修改 `accounts.csv` 時保留 header、略過完整公告區，避免資訊干擾。
- 主選單與子選單的錯誤提示、header、說明文字與切窗設定畫面都再做了一輪 UX 整理。

## 各範圍變更

### common
- `launch_delay` 的新預設值調整為 10 秒，同時保留對舊版 5 秒預設設定的向後相容與載入正規化。
- release build 現在會把 `version` 與 `releaseTime` 一起注入，啟動公告可直接顯示目前版本與 build 時間。

### multiboxing
- 新增並整理每帳號 `LaunchFlags` 設定流程，讓玩家更容易批次設定、取消與查看目前狀態。
- 單帳號啟動若該帳號已在執行中，會停留等待玩家閱讀訊息；批次啟動若所有帳號都已啟動，也會先顯示說明再回主流程。
- 進入區域選擇前，現在會先列出準備啟動的帳號清單，降低誤操作風險。
- 首次建立 `accounts.csv` 的流程已調整為先完成資料夾 / 檔案準備，再決定是否顯示完整公告區。
- 文件已補充目前 `LaunchFlags` 只提供兩種設定，且實測上 `-ns` 效果明確、`-lq` 仍偏向不確定。

### switcher
- 視窗切換設定頁重新整理目前狀態與提示文字，保留已保存設定的可見性。
- 已保存的切窗綁定現在可更穩定地在啟用 / 關閉之間切換，不必每次重設。

### docs / workflow
- README、多開使用文件、技術文件與 skill 說明持續對齊目前 CLI 行為與 mod / flag 描述。
- release workflow 現在要求同步維護版本與秒級 `releaseTime`，並把這份資訊顯示在啟動畫面。

## 升級注意事項
- 如果你習慣看啟動畫面確認版本，現在會額外看到 build 時間戳。
- 若這次是第一次執行或剛建立新的 `accounts.csv`，程式會優先引導你修改帳號檔，不會先塞滿完整公告內容。
- `mods` 啟動流程目前仍維持 `-mod <name> -txt`。
- `LaunchFlags` 目前仍只提供兩個可選項目；其中 `-lq` 是否真的有效，建議先自行實機驗證。
