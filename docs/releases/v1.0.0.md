# v1.0.0

## 摘要
- `d2r-hyper-launcher` 正式整理為 `v1.0.0` 穩定版發行。
- 這個版本整合了多帳號啟動、mod 選擇、視窗切換、首次使用引導與玩家導向文件，作為目前對外的正式版本基準。
- 同步整理版本顯示與 release 文件流程，讓玩家在 README 與 release note 都能直接看到目前最新版本。

## 重點
- 第一次執行會自動建立 `accounts.csv`，並引導玩家完成初始設定，不需要先手動整理檔案。
- 單帳號、批次啟動與離線模式都會先檢查 `D2R.exe` 路徑，避免直接撞進無效路徑錯誤。
- 批次啟動會先列出所有帳號的 `[已啟動]` / `[未啟動]` 狀態，只啟動尚未執行的帳號。
- README 現在會直接標示目前最新版本，並連到對應的 release note。

## 各範圍變更

### multiboxing
- 建立更完整的玩家使用流程：第一次執行自動建立 `accounts.csv`、支援 Excel 友善的 UTF-8 BOM、並在離開前自動開啟資料夾。
- 單帳號啟動會先檢查是否已經執行中，避免重複啟動同一個帳號，並補齊 mod 選擇流程。
- 批次啟動預設等待時間為 30 秒，降低短時間重複登入造成 Battle.net 連線被擋的風險。
- 批次啟動前會先預掃描帳號清單，明確列出每個帳號目前是否已啟動；若全部已啟動，就直接結束而不再詢問後續選項。
- 主選單狀態判斷使用 `accounts.csv` 裡的 `DisplayName` 對應 `D2R-<DisplayName>` 視窗標題，並已在介面與文件補上提醒。

### switcher
- 內建視窗切換功能，支援鍵盤、滑鼠側鍵與搖桿切換 D2R 視窗。
- README 已補充：`switcher` 只有在 launcher 持續開著時才會生效。

### 文件 / workflow
- README 重寫成以玩家實際上手順序為主：下載、雙擊執行、修改 `accounts.csv`、回到工具使用。
- README 注意事項已補充登入卡住、30 秒啟動間隔、`config.json` 不建議手動修改、免責聲明等重點。
- release workflow 改成固定產出中文 release note，並同步維護 README 的最新版本連結。
- 修正啟動畫面版本標頭，避免 release build 顯示重複的 `v` 前綴。

## 升級注意事項
- `v1.0.0` 是第一份正式 release note；目前最新正式版本請以 README 指向的 release 為準。
- 舊的 `v0.1.0` tag 仍保留在 git 歷史中；若要追查當時的程式狀態，請直接查看 tag。
- 若你依賴主選單的已啟動 / 未啟動狀態，請勿在遊戲仍開著時修改 `accounts.csv` 裡的 `DisplayName`。
- 如果你的 D2R 不在預設安裝位置，請在工具內使用 `p` 選擇正確的 `D2R.exe`。
