# v0.1.0

## Summary
- First tagged release of `d2r-hyper-launcher` for Windows D2R players.
- Establishes the core multiboxing launcher flow, first-run onboarding, and window switcher support.
- Packages the recent multiboxing UX fixes so players can launch accounts more safely with clearer feedback.

## Highlights
- Auto-create `accounts.csv` on first run, open the data folder, and keep Excel-compatible CSV output.
- Let players choose the D2R path from inside the launcher instead of hand-editing config files.
- Improve batch launch flow with clearer pending/running visibility, better delay behavior, and safer duplicate-launch protection.
- Document the launcher for player onboarding, troubleshooting, and release-safe Windows development workflows.

## Scope changes

### multiboxing
- Add safer first-run account setup with generated `accounts.csv`, folder-opening guidance, and UTF-8 BOM handling.
- Default batch launch delay to 30 seconds to reduce rapid reconnect pressure during repeated launches.
- Guard single-account, batch, and offline launch flows behind a valid `D2R.exe` path selection flow.
- Align single-account launch with batch launch by adding mod selection and blocking relaunch when that account is already running.
- Pre-scan batch launches, show explicit `[已啟動]` / `[未啟動]` account status lists, and skip extra prompts when nothing remains to launch.
- Warn players that status detection depends on `DisplayName` matching the current D2R window title.

### switcher
- Include the built-in window switcher workflow and document that switching only works while the launcher stays open.

### docs / workflow
- Rework the player quickstart to focus on download, first run, account setup, and main menu usage.
- Clarify common login troubleshooting, launch pacing guidance, and user-facing caution notes.
- Add repository guidance for Windows-safe validation and release handling.

## Upgrade notes
- This is the first tagged release, so there is no previous release tag to compare against.
- Existing players should re-check `accounts.csv` sample content and `DisplayName` values if they rely on running-status detection.
- If your D2R path is not the default install location, use the launcher’s `p` option to select `D2R.exe`.
