# v1.3.0

## 摘要
- 多開啟動流程新增每帳號預設畫質、預設登入區域與預設 mod，單帳號與批次啟動都可直接按 Enter 使用各帳號已保存的預設值。
- 啟動畫面現在會先列出每個待啟動帳號預設準備使用的 region / mod / 畫質，讓玩家在真正啟動前先看清楚這次會用什麼設定。
- Windows launcher 圖示與玩家文件同步更新，現在 repo 首頁、使用導覽與技術導覽都反映目前的實際行為。

## 重點
- **每帳號預設啟動設定**：`accounts.csv` 擴充為 8 欄，新增 `GraphicsProfile`、`DefaultRegion`、`DefaultMod`。按 Enter 會使用各帳號自己的預設，而不是套用單一全域值。
- **啟動流程統一**：線上啟動流程固定為 `region -> mod -> graphics`。手動輸入編號只覆蓋這次啟動，不會回寫帳號設定；畫質選單新增 `0`，代表本次不改動 `Settings.json`。
- **預設值安全處理**：若預設 region / mod / 畫質缺失或已失效，launcher 會明確擋下 Enter 並列出帳號，不再默默 fallback。若保存的 `DefaultMod` 或 `GraphicsProfile` 已不存在，也會先自動清空再視為未設定。
- **舊參數收斂**：舊版 `-lq` 不再作為每帳號畫質方案，會在載入帳號時自動清掉；每帳號畫質改由 `GraphicsProfile` 與 `Settings.json` 流程處理。

## 各範圍變更

### multiboxing
- 新增具名畫質設定檔保存與指派流程，讓不同帳號在啟動前可套用不同的 `Settings.json`。
- 新增每帳號預設登入區域與預設 mod 管理流程，支援依帳號或依設定來源批次指派。
- 單帳號與 `a` 批次啟動都會先顯示本次目標帳號的預設 region / mod / 畫質摘要，再讓玩家決定要直接吃預設還是手動覆蓋。
- 若玩家在畫質選單輸入 `0`，本次啟動就完全不套用畫質設定檔，保留目前磁碟上的 `Settings.json`。

### repo / build
- Windows 版執行檔圖示更新為新的 launcher branding，並調整成較適合 Windows 顯示的圓角輪廓。
- release build 持續透過 `-X main.version` 與 `-X main.releaseTime` 注入版本與時間，啟動公告可直接顯示目前版本資訊。

### docs / workflow
- `README.md`、`README.en.md`、多開使用導覽與技術導覽已同步到目前的 8 欄 `accounts.csv`、三段式啟動流程與帳號預設行為。
- 文件新增 Battle.net 跨區首次登入提醒：若帳號未先透過正常 Battle.net 流程登入過目標區域，直接用帳密切區可能看到被鎖訊息，先正常登入一次後即可恢復。
- CLI 測試策略調整為優先驗證邏輯、狀態與副作用，而不是把玩家文案綁成脆弱的精確字串比對。

## 升級注意事項
- 舊版 4 / 5 / 6 / 7 欄 `accounts.csv` **不需要手動搬資料**；launcher 載入時會自動補空值並保持向後相容，之後儲存時會寫回目前的 8 欄格式。
- 若你以前用 `LaunchFlags` 裡的 `-lq` 控制副帳號畫質，升級後請改用主選單 `g` 的帳號預設畫質流程。
- 若你在直接用帳號密碼切換 Battle.net 區域時看到被鎖訊息，請先透過 Battle.net 正常切換到該區域登入一次，再回來用 launcher 啟動。
- 若某個帳號保存的預設 mod 或畫質設定檔已不存在，按 Enter 使用預設值時 launcher 會先清空該設定並要求你重新選擇，避免誤用失效設定。
