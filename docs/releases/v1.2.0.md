# v1.2.0

## 摘要
- 新增多語系支援，玩家可在首次啟動或主選單中選擇繁體中文或英文介面。
- switcher 新增帳號篩選功能，可指定哪些帳號參與切窗循環、哪些排除。
- 修正 CLI 子選單巢狀導航問題，`h` 現在從任何深度的子選單都能正確回到主選單。

## 重點
- **多語系介面（i18n）**：首次啟動時會自動偵測系統語系並詢問確認；主選單提供 `l` 選項可隨時切換語言；語言設定儲存在 `config.json`。
- **Switcher 帳號篩選**：在 switcher 設定畫面新增「切換帳號設定」，可個別切換每個帳號的包含/排除狀態，或一鍵全部包含/排除；設定儲存在 `accounts.csv`。
- **Switcher 單帳號邊界保護**：只剩一個帳號包含在切換循環時，switcher 不會停止運作，而是永遠讓焦點落在那個帳號；排除全部帳號或只剩一個時都會顯示警告。
- **CLI 導航可靠性**：重構所有子選單使用統一的 `runMenu`，確保 `b`/`h`/`q` 無論在幾層巢狀中都由同一份程式碼處理，`h` 永遠回主選單不會只回上一層。

## 各範圍變更

### common
- 新增 `internal/common/locale` 套件，收錄繁體中文（`zh-TW`）與英文（`en`）所有使用者可見字串。
- 語言設定以 `language` 欄位存入 `config.json`；載入時若未設定會觸發首次語系選擇流程。
- 系統語系自動偵測：Windows LCID 對應 `zh-TW`；其他系統語系預設為 `en`。

### switcher
- 新增帳號篩選功能（`s → 2`）：可個別切換、全部包含或全部排除帳號；篩選狀態以 `ToolFlagSkipSwitcher` 位元旗標儲存在 `accounts.csv`。
- `ToolFlags` 欄位為新增欄；舊版沒有該欄的 `accounts.csv` 載入時預設 0（全部參與切換），完全向後相容。
- 切換循環只剩一個帳號時，直接讓焦點落在唯一帳號，不中斷 switcher 運作。
- 包含帳號數量為 0 或 1 時，儲存設定後顯示警告，並等待玩家按任意鍵確認，確保訊息不會被畫面刷新覆蓋。
- switcher 設定頁面的帳號清單與主選單帳號清單現在使用相同的顯示邏輯，排版一致。

### cli / workflow
- `menu.go` 新增 `runMenu` / `runMenuRead` 作為所有子選單 input loop 的統一架構；`ErrNavHome` 自動在巢狀呼叫中傳播，任何深度按 `h` 都能回到主選單。
- `cli_flags.go` 消除原本的 `goto` 敘述，改以巢狀 `runMenuRead` 表達「內層 b 返回外層」語意，程式碼意圖更清楚。

### docs / workflow
- 新增 `docs/switcher-usage-guide.md` 帳號篩選操作說明。
- `docs/switcher-technical-guide.md` 補充 `ToolFlags`、`ToolFlagSkipSwitcher` 與帳號篩選的技術說明。
- `AGENTS.md` 更新導航慣例：所有新子選單一律使用 `runMenu`，警告後立即繼續流程時使用 `showWarningAndPause`。

## 升級注意事項
- 首次執行 v1.2.0 時會詢問語言選擇（系統語系已偵測並顯示建議），選完即儲存，往後不會再詢問。可在主選單 `l` 選項隨時切換。
- `accounts.csv` 新增了 `ToolFlags` 欄位（整數型）。舊版 CSV 不含此欄，載入時預設 0（等同全部包含在切換循環），**無需手動修改**；第一次儲存帳號篩選設定後欄位就會自動寫入。
- 若你有客製化 `accounts.csv` 或直接編輯欄位，請注意 `ToolFlags` 現在是最後一個有效欄。
